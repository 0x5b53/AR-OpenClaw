# Aspri AR Clone Export

Portable export package for recreating Aspri AR in another engine.

## Included files
- `ASPRI_AR_MASTER_PORTABLE.md` - single-file portable profile
- `ASPRI_AR_CLONE_PROFILE.md` - identity, user preferences, and behavior rules
- `ASPRI_AR_TOOLSTACK.md` - memory/toolstack/workflow notes

## Recommended use
1. Start with `ASPRI_AR_MASTER_PORTABLE.md` if the target engine needs one compact profile.
2. Use the other two files if the target engine supports a richer layered setup.
3. If the target engine supports memory/context bootstrapping, also port durable knowledge separately.

## Notes
This export is intentionally kept clean and minimal so it can be pushed to a dedicated GitHub repo without dragging the whole working workspace with it.
